# Технический план и TODO по улучшению AI-анализа для `AlgorithmOne` v2

## Цель

Улучшить AI-слой для `backend\scanner\Algorithms\AlgorithmOne`, чтобы:

- у `AlgorithmOne` был свой отдельный prompt в папке алгоритма;
- `GeminiAnalyzer` продолжал использовать уже существующую инфраструктуру вызова Gemini API;
- AI-анализ для v2 стал более надёжным, предсказуемым и полезным для аудита решения алгоритма;
- уменьшить риск ложных или "литературных" ответов от LLM, которые плохо подходят для backend-логики.

## Контекст проблемы

Текущая реализация `GeminiAnalyzer` в целом рабочая, но имеет несколько системных слабостей:

- prompt больше похож на общий аналитический комментарий, чем на технический аудит решения v2;
- есть рассинхрон между данными v2 и prompt-переменными;
- ответ модели парсится через regex по человекочитаемому тексту, что хрупко;
- в prompt недостаточно отражены ключевые механики v2: gating, penalties, threshold evaluation, red flags;
- AI не ограничен достаточно жёстко по источнику истины и может "додумывать" футбольный контекст.

## Основные рекомендации

1. Превратить AI из "футбольного комментатора" в "аудитор решения алгоритма".
2. Передавать в prompt не только общие данные матча, но и внутреннюю логику v2.
3. Убрать рассинхрон между именами полей в v2 и шаблоне prompt.
4. Перейти на структурированный JSON-ответ от Gemini.
5. Разделить режимы анализа:
   - `audit_mode` для backend-валидации;
   - `explain_mode` для человекочитаемого текста, если он понадобится позже.
